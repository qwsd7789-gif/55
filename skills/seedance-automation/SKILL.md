# Seedance Automation

将即梦相关工作按固定 SOP 执行，目标不是“讲平台能力”，而是把一次次可复用的产出路径稳定下来：**收需求 → 整理素材 → 写提示词 → 提交前校对 → 记录任务 → 查结果 → 沉淀复用**。

## 核心原则

- 优先产出**可复用 SOP**，不是一次性聊天说明。
- 每次任务都要留下**可追溯记录**：参考图来源、提示词版本、关键参数、submit_id。
- 素材、提示词、任务号三者必须能互相对应。
- 用户没有明确要求时，默认采用**最稳的执行路径**，不要炫技。
- 如果当前任务只是“写提示词”，就不要假装已经提交。
- 如果当前任务已经提交过，回复时优先给出**submit_id + 素材路径 + 当前状态**。

## 适用任务

使用这个 skill 处理以下类型请求：

- “把这个即梦写成 SOP”
- “把这套能力固定下来”
- “以后按这个流程做即梦”
- “用这几张图在即梦跑一个视频”
- “给我整理参考图 + 提示词 + 任务号”
- “帮我查这个即梦任务结果”
- “把即梦流程沉淀成 skill / 能力包”

## 标准工作流

### 1. 识别任务类型

先判断用户要的是哪一种：

1. **SOP 固化**：把流程写成 skill / 文档 / 模板
2. **提示词生成**：根据主题和素材写提示词
3. **提交流程整理**：明确本次该怎么上传、怎么填
4. **结果追踪**：根据 submit_id 查结果
5. **案例沉淀**：把已跑过的案例写入 SOP

如果一个请求混合了多件事，按这个顺序拆开：
**先固化流程，再记录案例，再补查结果。**

### 2. 收集最小必要信息

至少确认下面几项；用户已给出就不要重复问：

- 目标：要生成什么视频
- 时长：4-15 秒；超长内容按分段/延长策略处理
- 比例：16:9 / 9:16 / 1:1
- 素材：图片 / 视频 / 音频 / 纯文本
- 风格：广告、电商、剧情、短剧、产品展示、空间展示等

如果用户只说“按上次那个来”，优先回溯并复述：

- 参考图目录
- 取了几张图
- 当前 submit_id
- 上次已确定的主题或风格

### 3. 整理参考素材

把参考素材整理成可追踪形式：

- 记录原始来源目录
- 记录实际使用了哪些文件
- 明确每张图的用途：主体、场景、质感、首帧、尾帧
- 如果素材很多，先筛成“本次实际提交集合”

推荐记录格式：

```text
项目名：美容院
素材来源目录：D:\seedance 用参考图\美容院
实际使用：4 张
- ref1.png：门头/外景
- ref2.png：接待区
- ref3.png：护理区
- ref4.png：细节氛围
```

### 4. 生成提示词

提示词必须满足：

- 可直接复制到即梦
- 中文
- 明确时间顺序
- 明确参考用途
- 明确镜头、动作、空间、氛围
- 必要时显式写禁止项（字幕、水印、logo、畸形等）

优先采用以下结构：

```text
[主体/空间] + [动作或镜头推进] + [环境与光线] + [风格质感] + [参考图用途] + [禁止项]
```

如果是多参考图场景，明确写：

```text
@图片1 为外立面参考，@图片2 为前台空间参考，@图片3 为护理区空间参考，@图片4 为氛围与细节参考。
```

### 5. 提交前校对

在真正提交或交付用户前，检查：

- 时长是否在即梦允许范围内
- 图/视频/音频数量是否超限
- @引用是否一一对应
- 是否误用了写实真人脸素材
- 是否把“参考”写成了“编辑”
- 是否遗漏比例、时长、风格、声音要求

### 6. 记录提交信息

只要发生了提交，就必须记录最少这几项：

```text
项目名：
平台：即梦 / Seedance / Dreamina
素材目录：
实际使用素材：
提示词版本：
时长 / 比例：
submit_id：
提交时间：
备注：
```

如果用户问“是不是用这个文件夹里的图提交的”，优先按这个格式直接回答：

```text
已用这个目录里的图提交：<路径>
实际取用：<N> 张
submit_id：<任务号>
```

### 7. 查结果与回报

查结果时，回复优先级如下：

1. 当前状态（排队中 / 生成中 / 已完成 / 失败）
2. submit_id
3. 对应素材目录
4. 如已完成，给结果文件或下一步建议

不要只回“我去查一下”；能给状态就直接给状态。

### 8. 沉淀成可复用案例

对成功案例，补一段“案例卡片”，便于以后复用：

```text
案例名：美容院空间展示
目标：用 4 张美容院参考图生成空间氛围视频
素材目录：D:\seedance 用参考图\美容院
素材数量：4
submit_id：b62f8e713b89c002
可复用点：多图空间展示、门头到室内切换、生活方式氛围包装
```

## 输出模板

### A. 用户要 SOP

按这个结构输出：

1. 目标
2. 适用场景
3. 输入要求
4. 操作步骤
5. 校对清单
6. 记录模板
7. 一个真实案例

### B. 用户要本次提交说明

按这个结构输出：

```text
已按以下素材提交：
- 素材目录：...
- 实际使用：... 张
- submit_id：...
- 当前可继续动作：查结果 / 改提示词 / 追加新版本
```

### C. 用户要复用能力

优先落成 skill，命名用小写短横线；若用户给了中文名，则：

- skill 文件夹名：`seedance-automation`
- 对外展示名可写：`seedance自动化`

## 何时读取扩展参考

- 需要写更复杂的即梦视频提示词时，参考现有全量提示词技能：`seedance-提示词`
- 需要查看更细的 SOP 模板和案例结构时，读取 `references/sop-template.md`

## 注意事项

- 不要把“平台介绍”写得过长，重点是可执行 SOP。
- 不要只记录 submit_id，不记录素材来源。
- 不要只记录素材来源，不记录最终用了哪几张。
- 如果用户明确说“固定能力”，优先生成 skill 或标准文档，而不是只在聊天里解释。
